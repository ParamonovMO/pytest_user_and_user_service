[![Python](https://img.shields.io/badge/Python-3.10%2B-blue)](https://python.org)

> **Этот репозиторий — не просто код, а демонстрация моего подхода к написанию промышленных автотестов на Python.**  
> Здесь я показываю, как строить чистые, читаемые и информативные тесты, используя лучшие практики и современные инструменты.

## 👀 О чём этот проект?

Я написал простой сервис для управления пользователями (`UserService`) и **полностью покрыл его тестами**. Цель проекта — не показать сложный продакшен-код, а **показать архитектуру тестов**, работу с фикстурами, параметризацией и создание отчётов, которые не стыдно показать команде.

**Здесь вы увидите:**
*   Как использовать **фикстуры** (`fixtures`) для подготовки данных вместо громоздких `setUp`.
*   Как с помощью **параметризации** (`@pytest.mark.parametrize`) проверить десятки сценариев в одном методе.
*   Как тестировать **исключения** и граничные условия.
*   Как превратить консольный вывод `pytest` в красивый **Allure-отчёт** с графиками и пошаговыми скриншотами выполнения.

## 🛠 Технологический стек и чему я научился

| Инструмент | Зачем я его использовал? |
|------------|--------------------------|
| **Pytest** | Основной фреймворк. Научился писать `assert`'ы без лишнего кода. |
| **pytest-cov** | Измеряю покрытие кода, чтобы видеть "слепые зоны". |

## 🚀 Как это запустить и посмотреть магию?

Если хотите увидеть всё своими глазами, следуйте инструкции ниже.

### 1. Подготовка (один раз)
```bash
# Клонируем репозиторий
# Создаем виртуальное окружение и ставим всё необходимое
python -m venv venv
pip install -r requirements.txt

# Просто запустить тесты с подробным выводом
pytest -v

# Собираем данные для отчёта
pytest --alluredir=./allure-results

# Смотрим отчёт в браузере (откроется сам)
allure serve ./allure-results

# Увидим процент покрытия и какие строки я пропустил (спойлер: 100% 😉)
pytest --cov=user_service --cov-report=term-missing

📸 Пример того, что вы увидите в Allure
Вместо скучной "простыни" в консоли, вы получите дашборд:

Behaviors: Тесты сгруппированы по функциям ("Добавление пользователя", "Обновление email").

Suites: Техническая группировка по файлам.

Пошаговые логи: На каждый тест можно нажать и увидеть, что происходило внутри (какой был запрос, какой пришёл ответ).
